# Gouvernance

## 1. Modèle actuel

TCP/UP est actuellement maintenu par son fondateur / mainteneur.

À ce stade, la gouvernance reste légère afin de permettre une clarification rapide du protocole et une itération publique.

## 2. Comment les évolutions sont décidées

Les évolutions peuvent être proposées via :
- des issues GitHub,
- des pull requests,
- des cas d’usage documentés,
- des critiques sur les cas limites,
- des suggestions d’amélioration de formulation.

Le mainteneur examine puis accepte, refuse ou reporte les propositions.

## 3. Principes de décision

Les évolutions sont évaluées selon :
- la clarté,
- la transparence,
- la cohérence avec le protocole,
- l’utilité dans des situations réelles de publication,
- la prudence dans les cas ambigus.

## 4. Politique de versionnement

- Version majeure : changement structurel du protocole
- Version mineure : clarification, ajout d’exemples, évolution non cassante
- Version corrective : correction de formulation, faute ou mise en forme

## 5. Orientation future

L’objectif à long terme est d’évoluer vers une gouvernance plus ouverte, pouvant inclure :
- des contributeurs externes,
- des experts de domaine,
- des relecteurs éditoriaux,
- une consultation publique sur les révisions importantes.

## 6. Note de gouvernance

TCP/UP reste une initiative déclarative de transparence. La gouvernance du projet n’en fait pas une autorité de certification.
